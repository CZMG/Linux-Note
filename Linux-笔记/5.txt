RHEL7下 第二种破解root密码的方式（推荐）
内核信息尾部加   rd.break      
mount -o remount,rw /sysroot/   重新挂载并要求sysroot可以读写
chroot /sysroot/    切换根
passwd  修改密码
load_policy -i   加载selinux
chcon -t shadow_t /etc/shadow   修改shadow 的上下文
touch ./autorelabel    重建selinux标签

RHEL7中的grub的修复
通过光盘引导
chroot /mnt/sysimage  切换根
grub2-install /dev/sda   安装grub
================================================
RHEL7中内核等引导文件的修复
chroot /mnt/sysimage 
mount /dev/cdrom /media
rpm -ivh /media/Packages/kernel*****    --force
安装完成后，系统中会生成内核和虚拟文件系统。
grub2-install /dev/sda   
grub2-mkconfig -o /boot/grub2/grub.cfg
================================================
RHEL 7 添加Grub加密
在/etc/grub.d/00_header的末尾添加如下内容：
cat << EOF
set superusers="tom"
password tom redhat
EOF
最后执行更新grub.cfg操作
grub2-mkconfig -o /boot/grub2/grub.cfg

网络的基本配置

静态配置网络：
IPADDR:
NETWORK:
GATEWAY:

硬件检测
cat  /proc/cpuinfo      查看CPU的信息
lspci                            查看所有的PCI信息
cat  /proc/meninfo   查看存储的信息
cat /proc/bus/usb/devices   查看所有USB接口设备的信息
cat /proc/bus/input/devices   查看所有输入设备的信息
fdisk  -l                    查看硬盘的信息
df  -h                       查看硬盘的信息
uname -a 
cat /proc/version 
lsb_release -a         查看产品发布信息（lsb是linux的一个核心标准）
lsblk                      列出系统中的所有块设备信息
cat /proc/interrupts   查看设备中所有的IRQ信号
dmidecode -t bios      查看BIOS信息
pstree -p |grep udev   查看udev这个daemons（守护进程），自动设备的名称和定义设备如何使用。它在/etc/rc.d/rc.sysinit脚本后开始工作。/etc/udev/rules.d/下，存放了系统的规则文件。
pstree -p |grep hal      hal这个daemon是实现程序和设备之间交流的，告诉程序如何使用设备。

软件安装
debain: 
	/etc/apt/source.list
	sudo apt-get  update   更新源
	sudo  apt-get upgrade  更新系统（保留原系统）
	sudo apt-get dist-upgrade  更新系统（删除原系统）
	sudo  apt-get   autoremove自动删除一些不常用的软件包

源码编译安装
	优点：定制性高
	缺点：耗时长
	./configure   定制环境。   --prefix=路径，定制安装路径。（有makefile，则说明该软件包的安装路径已被定义）
	make   编译
	make install   安装

rpm包安装（debain  dpkg）
	优点：简单方便
	缺点：功能无法定制，存在包依赖
	rpm 
		-qa  查看系统中所有的rpm软件包
		-qa | gerp kernel     查看系统中包含kernel字符的软件包
		-ql    kernel*    查看某个包安装的位置
		-qc    kernel*   查看某个软件包的配置文件的安装位置
		-qd    tree       查看某软件的帮助文档
		-qf    /usr/bin/rz     反向查看某程序的名称
		-qlp   软件包名      安装前查看某软件包将会安装的位置
		-K    软件包名       查看安装的软件包是否存在数字签名
		-ivh   软件包名      安装软件包
		-Uvh  软件包名    U表示更新rpm包，如果包没有安装过，那就安装，如果装过，先卸载，重新安装当前版本。（升级内核类的不要使用这种更新方式）
		-e      软件包名     卸载软件包
		
yum安装
	优点：尽可能解决包依赖、方便
	缺点：存在版本依赖
	源位置
		rhel：/etc/yum.repos.d/
		debain、ubuntu:/etc/apt/source.list.d/
	源容器的语法：
		[源服务器名称]
		name=源服务器的名字，类似注释
		mirrorlist=
		#baseurl=
		enabled=1
		gpgchech=1(是否启用gpg校验,0不需要，1需要)
		gpgkey=file://路径
	yum安装
		yum install name    安装软件包
		yum remove name   卸载软件包
		yum search   name   检索软件包
		yum clean all           清除yum缓存
		yum list name         列出软件包
		yum check-update   检查系统是否需要更新
		yum update/upgrade  更新系统 
		
dnf安装
	dnf --list =yum list
	dnf search tree =yum search tree   检索软件包
	dnf provides /usr/bin/rz=yum   provides /usr/bin/rz    根据程序的指令来获取程序安装包的名称
	dnf whatprovides /usr/bin/rz
	dnf list available    列出当前系统能够安装的软件包
	dnf install/remove  软件包名
	dnf autoremove   自动删除一些不常用的软件包
